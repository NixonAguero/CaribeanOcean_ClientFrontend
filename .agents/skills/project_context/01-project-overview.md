## General Requirements
- **Architecture:** N-layer pattern (Presentation → Business Logic → Data)
- **Methodology:** SCRUM with Trello (board name: year + GroupName)
- **Version control:** Git with Conventional Commits
- **CI/CD:** DevOps pipeline
- **AI:** Responsible use for prototypes, specific layers, automated tests
- **AI model:** Train model with synthetic data to predict hotel reservations
- **ScrumMaster rotation:** Each team member must lead one sprint (mandatory)
- **Biweekly reports:** What was done, what will be done next, blockers

## ScrumMaster Responsibilities per Sprint
- Upload completed and planned user stories
- Send all sprint deliverables
- Negotiate with product owner (professor) on issues
- Compile individual team reports
- Send meeting agenda to professor by 11:59pm day before delivery

## Retrospective Meeting (end of each sprint)
- Demo what was implemented
- Explain what went well, what went poorly, what to improve
- All team members must participate and explain their work
- Professor can ask technical questions to any member
